# React Calculator

I made this repository to learn the basics of React...

## Stack

- React
- Zustand
- Immer
- Tailwind

# What I learned

- Keeping state management to what is only necessary
- The power of the "resetKey" pattern.
- Separating logic from state
- Keeping a single source of truth

## General thoughts

This brings me back to when I made a vanilla JS calculator for TOP, and simultaneously loved and hated the experience. But now I am working with React and Typescript, the experience is actually much faster? I don't have to throw for loops everywhere, just put things together. It feels ILLEGAL. It's less code, and whatever thinking I have to do is more about wiring logic with state and display properly. I understand why businesses want a React guy, or girl.

Yes, I made this without a tutorial. And it is still devastating it took me in about a third of the time it took me to build the vanilla calculator.
